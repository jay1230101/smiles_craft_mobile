1. How to install (Android)

  1. Download the APK file (link shared separately on WhatsApp / email).
  2. On your Android phone, open Files → Downloads → tap the APK.
  3. If prompted "For your security, your phone isn't allowed to install unknown apps":
    • Tap Settings → enable Allow from this source → go back and tap Install.
  4. After install, find Smile Craft on your home screen and open it.

3. What to test

3.1 Splash + Launch

  • App opens to a white splash screen showing the Smile Craft logo.
  • After 1–2 seconds, the splash transitions cleanly into the Login screen (or Dashboard if already logged in).
  • App icon on the home screen shows the blue Smile Craft swoosh on white background — not a generic Expo icon.

3.2 Login Screen

  • Page shows: Smile Craft logo, "Welcome Back!" title, subtitle, Email + Password fields, Remember me checkbox, Forgot Password? link, Login button.
  • Tap Login with empty fields → red error messages appear under each field ("Email is required", "Password is required").
  • Type an invalid email (e.g. "abc") → "Enter a valid email" appears on blur.
  • Type a valid email + wrong password → tap Login → "Invalid email or password" appears in red under the form.
  • Tap the eye icon on the password field → password becomes visible; tap again → hidden.
  • Tap Remember me → checkbox fills; tap again → unchecks.
  • Tap Forgot Password? → navigates to forgot password screen.
  • Enter correct credentials → tap Login → app opens to Dashboard.

3.3 Forgot Password Screen (view only — skip the full flow)

For this round, only verify the Forgot Password screen renders correctly. Don't submit the form or test the rest of the reset flow — the end-to-end reset still routes through the web app and the backend mobile endpoint isn't ready yet.
  • From Login, tap Forgot Password? → navigates to the Forgot Your Password screen.
  • Page shows: back arrow, title "Forgot Your Password", subtitle, email input field, "Send Reset Link" button, "Didn't receive the email? Resend Link" footer.
  • Layout matches the designer's Figma (spacing, typography, colors).
  • Tap the back arrow → returns to Login screen.

  • All 5 tabs visible: Home / Register / Calendar / Billing / Reports.
  • Each tab has a brand icon.

  • Inactive tabs: icon and label in gray, label in regular weight.

3.8 Logout Flow

  • On Dashboard, tap the logout icon (top-right, next to bell).
  • Native confirmation dialog appears with "Log out" as title and "Are you sure you want to log out?" as message.
  • Tap Cancel → dialog dismisses, stays on Dashboard.
  • Tap the logout icon again, then Log out (in red on iOS, default on Android) → returns to Login screen.
  • Try to navigate away from login (back button on Android) → should stay on login (no access without auth).
  • Log in again with correct credentials → lands back on Dashboard.

3.9 Responsive Layout

Test on different physical phones if possible:

  • No content cut off on any phone size.
  • Status bar and bottom gesture indicator don't overlap content.
  • Tab bar labels readable on narrow phones (not wrapping or truncating awkwardly).
  • All buttons easily tappable.

4. Known limitations (please don't report these)

The following are planned but not yet built — these are expected behaviors, not bugs:
  • Bell icon on Dashboard doesn't open notifications (UI placeholder, full notifications come in a later milestone).
  • + button on All Appointments doesn't open a "new appointment" form (Calendar M3 work).
  • Tapping an appointment card doesn't open a detail view (M3).
  • Calendar live sync — when the web app changes an appointment, the mobile won't visibly update until you exit and re-enter the screen.
  • Update Password screen is currently reachable only by manual navigation — the email reset link still goes to the web. End-to-end mobile password reset needs a backend endpoint not yet built.
  • Register / Billing / Reports tabs are placeholders ("coming soon" screens) — to be built in M3 and M4.
  • Resend Link on Check Email screen does its job silently — no toast / confirmation appears yet.
  • App name in some places shows "smilescraft-mobile (Dev)" or similar — this is normal for non-production builds and will say just "Smile Craft" in store builds.
  • Dashboard, All Appointments, and Calendar currently show sample data (Michael Chen, Emily Rodriguez, etc.) — this is intentional for a consistent demo experience and will be swapped to live data before final release.

5. What to report

Please report anything you observe that:

  • Causes the app to crash, freeze, or restart.
  • Shows text or images cut off, overlapping, or going off-screen.
  • Has buttons that don't respond when tapped.
  • Looks visibly different from the designer's Figma (not matching colors / spacing / typography).
  • Contains typos or grammar mistakes.
  • Behaves inconsistently between two test runs (e.g., worked once, didn't work next time).
  • Is confusing or unclear from a user perspective.
